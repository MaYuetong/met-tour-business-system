# 技术决策：邮件方案

## 双轨机制

### 轨道 1：Nodemailer + Gmail SMTP
- **用途**：向用户发送交易邮件（预约确认、问卷回顾）
- **账号**：`met.euart@gmail.com`
- **模板**：`lib/email.ts → buildHtml()` / `buildSurveyEmailHtml()`
- **风格**：Met 博物馆品牌 HTML 邮件

### 轨道 2：Formspree
- **用途**：向管理员（Yuetong）发送即时通知
- **Endpoint**：`https://formspree.io/f/xbdwojzk`
- **无需 API Key**，直接 POST JSON
- **触发场景**：新预约、参观前问卷、参观后问卷
- **函数**：`lib/email.ts → notifyAdminFormspreeRaw(subject, data)`

## 为什么用双轨？
- Nodemailer 适合精美 HTML 模板，但配置复杂
- Formspree 极简，0 配置，适合管理员通知
- 两者互补，避免单点故障

## 注意事项
- Vercel 无服务器函数中，所有 `await` 必须在 `return` 前完成
- 邮件发送失败不应导致主流程报错（catch 后 console.error 即可）
- Gmail SMTP 需开启「应用专用密码」，存于 Vercel 环境变量
