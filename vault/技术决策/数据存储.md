# 技术决策：数据存储

## 方案：Vercel KV（Upstash Redis）

### 原因
- 与 Vercel 原生集成，零配置部署
- 无服务器友好（无连接池问题）
- 免费额度满足早期业务量

### 数据结构
所有数据以 JSON 文件形式存储于 KV，key 为文件名：

| Key | 类型 | 说明 |
|-----|------|------|
| `bookings.json` | `BookingRecord[]` | 所有预约 |
| `pre-surveys.json` | `PreSurveyRecord[]` | 参观前问卷 |
| `post-surveys.json` | `PostSurveyRecord[]` | 参观后问卷 |
| `reviews.json` | `ReviewRecord[]` | 访客评价 |
| `referrals.json` | `ReferralRecord[]` | 推荐码 |
| `guides.json` | `GuideRecord[]` | 讲解员 |

### 读写函数（`lib/db.ts`）
```typescript
read<T>(key)    // 读取，不存在返回 []
write<T>(key, data)  // 覆盖写入
```

### 注意事项
- 本地开发用 `.env.local` 配置 KV 连接
- 生产环境变量在 Vercel Dashboard 设置
- KV 操作有延迟（~10ms），Vercel 函数超时默认 10s
